## 7. 会话总结

- **本次开发最成功的决策**：① 文档整理时建立「docs/README 为文档中心 + PROJECT-STATUS 为状态汇总 + 根 README 为项目入口」的三层结构；② 保留 PROJECT-STATUS-TEMP 重定向而非直接删文，既统一入口又保留历史可追溯；③ 删除 server 时同步修改 .cursor/plans 中的基线描述，避免失效链接与误导。
- **本次开发最大的教训**：无显著教训；若提前约定会话总结由系统提供轮数/字符数，可减少估算量。
- **下次类似任务可复用的经验**：① 删除「疑似未使用」目录前，先查根 pom 的 modules、再查代码与配置中对路径/包名的引用，可快速判断是否安全删除；② 文档大整理时先定索引结构（分类 + 推荐阅读顺序），再补写或改写单篇，效率高于逐篇改写再事后归纳；③ 涉及 Git「从远程删除」类操作，单独写步骤文档并注明「一次性」与「历史重写风险」，便于执行与归档。
